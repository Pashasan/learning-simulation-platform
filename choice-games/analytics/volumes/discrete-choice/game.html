What's the fundamental problem with linear regression for binary choices?
It predicts values outside 0–1, which aren't valid probabilities
It runs too slowly on binary outcome data
It requires many more data points than logistic regression
It can only handle continuous predictor variables
Chapter 1

The Sigmoid Solution

Logistic regression uses the sigmoid function to squash any input into a probability between 0 and 1. As a product becomes more attractive (higher utility), the probability of choosing it approaches — but never reaches — 1.
As utility increases far above 0, the logistic curve approaches:
Infinity — there's no upper bound on the output
1.0 — but never quite reaches it (diminishing returns)
0.5 — it always returns to the midpoint
0 — high utility means low probability

Exploring the S-Curve

The logistic function is: P = 1 / (1 + e−utility). The inner part (β₀ + β₁X₁ + ...) is called the utility.

Diminishing Returns

Adding the same +0.5 utility has a very different effect depending on where you start. Compare:
Starting at 50%  (utility = 0)
50%
 
Starting at 95%  (utility = 2.94)
95%
 
A product already has 95% choice probability. Adding a feature increases utility by 0.5. The probability will:
Jump to 100% since any extra utility pushes it to the max
Increase only slightly — diminishing returns near the extremes
Stay exactly the same — the curve is flat at 95%
Decrease because of model saturation
