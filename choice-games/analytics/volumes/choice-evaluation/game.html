Chapter 1

Precision vs. Recall

Two key metrics built from the confusion matrix:

Precision = TP / (TP + FP) — "When we predict Buy, how often are we right?"
Recall = TP / (TP + FN) — "Of actual buyers, how many did we catch?"
Precision answers which business question?
What percentage of all 200 test consumers did we classify correctly?
Of the consumers we targeted as buyers, what fraction actually bought?
Of the actual buyers, what fraction did the model identify?
How many false positives does the model produce overall?

Chapter 2

The Business Trade-off

BeatBox's Dilemma

BeatBox is planning production quantities. Each unit costs £35 to make. Overproduction (making units nobody buys) is their biggest risk.

  • False Positive = predict "Buy" but they don't → wasted production
  • False Negative = predict "Skip" but they would buy → missed sale
To minimize overproduction, which metric should BeatBox prioritize?
Precision — when we predict "Buy," we want to be right
Recall — make sure we catch every potential buyer
Accuracy — maximize overall correctness
F1 Score — balance everything equally

Chapter 4

Beyond Binary

What if consumers choose from three or more products?

Lena Product Manager

"We're expanding the lineup to three tiers: Budget, Mid-Range, and Premium. The binary model won't cut it."

The multinomial logit extends binary choice to N alternatives using the softmax function. Each product gets a utility, and probabilities are computed as:
P(i) = exp(Vi) / Σ exp(Vj)

In the multinomial logit, what must be true about the probabilities across all products?
They must sum to exactly 1 (100%)
Each product must get an equal share (1/N)
The top product must always have P > 50%
Probabilities can be negative for disliked products
